- DEMO_CODE: Shows Proxy get/set trap concept mapped to Vue's reactive().
- Explanation: Vue wraps objects in Proxy — `get` trap records which effect (component/watcher) accessed the property, `set` trap triggers those specific effects. Automatic fine-grained tracking. React has no tracking — state is immutable, you call setter explicitly, entire component re-renders.
- Diagram: "Proxy-based Tracking" — flow: `reactive({})` -> `Proxy` -> two branches: `get: Track dependency` and `set: Trigger effects` -> `Re-render affected`. Use marker id="arrow-reactivity-vue".

This demonstrates Vue's automatic optimization.
- DEMO_CODE: Shows parent + child where child doesn't re-render on parent state change.
- Explanation: Vue components only re-render when their own reactive dependencies change. No `memo()` needed. React re-renders ALL children when parent state changes unless wrapped in `React.memo()`. Vue's compiler + reactivity system provides this optimization for free.

- DEMO_CODE: Shows React component where entire function body re-executes on setState.
- Explanation: React's reconciler (Fiber) re-executes the component function on every state change, then diffs the returned JSX tree against the previous tree. No compile-time optimization like Vue. React depends on runtime diffing of the full subtree.
- Diagram: "React Full Subtree Re-render" — flow: `setState()` -> `Component fn()` (highlighted — "entire function re-executes") -> `New JSX Tree` -> `Reconciler diffs all nodes` -> `DOM Update`. Use marker id="arrow-rendering-react".

**ReactivityInDepth.tsx (DEEP-02):**

- Live demo: Show useState where calling setter replaces the entire value. Log panel (bg-slate-900 text-green-400) shows "setState called -> full re-render" each time. No automatic tracking — you must explicitly call the setter.
- DEMO_CODE: Shows useState pattern with no automatic dependency tracking.
- Explanation: React state is immutable — `useState` returns a snapshot and a setter function. No Proxy, no tracking. When setter is called, the entire component re-renders. Derived values need explicit `useMemo` with dependency array. Vue's Proxy-based system tracks automatically.
- Diagram: "Immutable State + Setter" — flow: `useState(init)` -> `[value, setter]` -> `setter(newVal)` -> `Component re-render (full)` -> `All children re-render`. Use marker id="arrow-reactivity-react".

**RerenderOptimization.tsx (DEEP-03):**

- Live demo: Parent with `useState(0)`, child component WITHOUT `React.memo`. Show render counter in child proving it re-renders every time parent state changes. Then add a second child wrapped in `React.memo` — show its render counter does NOT increment. Demonstrates the need for manual optimization.
- DEMO_CODE: Shows React.memo, useMemo, useCallback usage for optimization.
- Explanation: React re-renders all children by default on parent state change. To prevent: wrap child in `React.memo()` (skips if props unchanged), use `useMemo()` for expensive computations, `useCallback()` for stable function references. Vue does this automatically via its reactivity system.
- Diagram: "Manual Optimization" — tree: `Parent (state changes)` -> arrow to `Child A (no memo)` labeled "re-render" + arrow to `Child B (React.memo)` labeled "props same?
