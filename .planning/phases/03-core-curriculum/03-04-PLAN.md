**ComposablesHooks.tsx (REUS-01):**
- Define a custom hook `function useCounter(initial = 0)` that returns `{ count, increment, decrement, reset }` using useState.
- Demo: Use the hook in the main component. Show counter buttons. Point out the reusability pattern.
- DEMO_CODE: Show the custom hook definition and usage
- Explanation: Vue composables (functions returning reactive state) map directly to React custom hooks. Both follow the same "extract stateful logic" pattern. Vue composables use `ref()`, React hooks use `useState()`. Naming convention: Vue `useX()` / React `useX()` — identical.

**CustomDirectives.tsx (REUS-02):**
- React has NO equivalent of Vue directives. Demo shows the React alternative: a custom hook `function useAutoFocus(ref: React.RefObject<HTMLInputElement>)` that focuses on mount via useEffect.
- Demo: Input that auto-focuses when shown via toggle.
- DEMO_CODE: Show useEffect-based auto-focus hook vs Vue's `v-focus` directive
- Explanation: Vue custom directives (`app.directive('focus', { mounted(el) { el.focus() } })`) attach behavior directly to DOM elements in templates. React has no directive system — the equivalent is a custom hook or HOC. Directives are Vue-only syntactic sugar for DOM manipulation.

**Plugins.tsx (REUS-03):**
- Demo: Show React's Context/Provider pattern as the equivalent of Vue's `app.use()` plugin system. Create a simple `NotificationContext` with `createContext` + Provider. A child component uses `useContext` to show/trigger notifications.
- DEMO_CODE: Show createContext + Provider pattern
- Explanation: Vue plugins (`app.use(plugin)`) can add global methods, components, directives, and provide/inject values. React has no plugin system — the closest equivalent is Context/Provider pattern for DI, and custom hooks for shared logic. Vue plugins are more flexible (access the app instance), React is more explicit.

- Explanation: Vue has built-in `<Transition>` component that auto-applies CSS classes (`v-enter-active`, `v-leave-active`). React has no built-in transition — use CSS transitions + className toggle, or libraries like framer-motion. Vue's approach is more declarative.

- Explanation: Vue's `<TransitionGroup>` handles list enter/leave/move animations automatically with CSS classes. React requires manual CSS transitions or a library. Vue's built-in is significantly simpler for list animations.

**KeepAlive.tsx (BTIN-03):**
- Demo: Two "tabs" (Tab A with a counter, Tab B with a text input). Switching tabs without KeepAlive resets state. Show the manual state preservation approach: lift state up to parent, pass down as props.

- DEMO_CODE: Show manual state preservation pattern (lifted state)
- Explanation: Vue's `<KeepAlive>` caches component instances — switching back restores exact state without lifting. React has no built-in equivalent — must lift state to parent or use a state management library. Vue's approach preserves internal state automatically.

**Teleport.tsx (BTIN-04):**

- DEMO_CODE: Show createPortal usage
- Explanation: Vue's `<Teleport to="body">` and React's `createPortal(children, container)` do the same thing — render content in a different DOM location. Vue's syntax is more declarative (in template). React's is a function call. Both preserve event bubbling through the component tree.
