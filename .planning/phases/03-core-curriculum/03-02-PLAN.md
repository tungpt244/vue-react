**ConditionalRendering.tsx (ESS-04):**

- Demo: A toggle button that shows/hides content. Three sections demonstrating:
  1. `&&` operator — show a message when boolean is true
  2. Ternary `? :` — toggle between two different messages
  3. Early return pattern — component that returns different JSX based on a `status` state ('loading' | 'error' | 'success')
- State: `const [show, setShow] = useState(true)`, `const [status, setStatus] = useState<'loading' | 'error' | 'success'>('success')`
- DEMO_CODE: Show React conditional patterns (&&, ternary, early return)
- Explanation: Vue has `v-if`/`v-else`/`v-show` directives in template. React uses JS expressions directly in JSX: `&&`, ternary, early return. `v-show` toggles CSS display, React equivalent is `style={{ display: show ? '' : 'none' }}`.

**ListRendering.tsx (ESS-05):**

- Demo: A list of items with add/remove. Input + button to add items. Each item has a delete button. Show the `.map()` + `key` pattern.
- State: `const [items, setItems] = useState([{ id: 1, text: 'Learn Vue' }, { id: 2, text: 'Learn React' }])`, `const [newItem, setNewItem] = useState('')`, `const [nextId, setNextId] = useState(3)`
- DEMO_CODE: Show `.map()` with key prop
- Explanation: Vue uses `v-for="item in items" :key="item.id"` directive. React uses `items.map(item => <li key={item.id}>...)`. Both need unique `key` for efficient DOM diffing. Vue's `v-for` also provides `(item, index)` — React's `.map` gives the same via callback second arg.

**EventHandling.tsx (ESS-06):**

- Demo: Three sections:
  1. Click counter with onClick
  2. A link that prevents default navigation (show `e.preventDefault()`)
  3. A keydown handler on input (Enter key logs a message)
- State: `const [count, setCount] = useState(0)`, `const [log, setLog] = useState<string[]>([])`
- DEMO_CODE: Show onClick, e.preventDefault(), onKeyDown
- Explanation: Vue uses `@click`, `@click.prevent`, `@keydown.enter` modifiers. React uses `onClick`, manual `e.preventDefault()`, `onKeyDown` with `if (e.key === 'Enter')`. Vue modifiers are syntactic sugar — React requires explicit JS.

**FormBindings.tsx (ESS-07):**

- Demo: A form with text input, select dropdown, and checkbox — all controlled components. Show live preview of form values below.
- State: `const [name, setName] = useState('')`, `const [framework, setFramework] = useState('react')`, `const [agree, setAgree] = useState(false)`
- DEMO_CODE: Show controlled input, select, checkbox patterns
- Explanation: Vue has `v-model` which is two-way binding sugar. React requires explicit `value` + `onChange` for each input (controlled components). Vue's `v-model` on `<select>` works the same as input — React's `<select value={x} onChange={...}>` is equivalent.

**TemplateRefs.tsx (ESS-09):**

- Demo: An input field with a "Focus" button. Clicking the button programmatically focuses the input using `useRef`.
- State: `const inputRef = useRef<HTMLInputElement>(null)`, `const [value, setValue] = useState('')`
- Import `useRef` from 'react'
- DEMO_CODE: Show useRef creation and .current access
- Explanation: Vue uses `ref="myInput"` in template + `const myInput = ref<HTMLInputElement>()` in setup. React uses `useRef<HTMLInputElement>(null)` + `ref={inputRef}`. Both access via `.value.focus()` (Vue) and `.current.focus()` (React). Key difference: Vue template refs are reactive, React refs are not.

All files: Button styling `className="px-3 py-1 bg-blue-500 text-white rounded hover:bg-blue-600 text-sm"`. Input styling `className="border border-slate-300 rounded px-2 py-1 text-sm"`. Code block uses `p-4` (NOT p-3).
