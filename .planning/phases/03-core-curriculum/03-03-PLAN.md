**Props.tsx (COMP-01):**

**EventsCallbacks.tsx (COMP-02):**

- Define inline child `function ChildButton({ onIncrement }: { onIncrement: () => void })` with a button that calls the callback.
- Demo: Parent has counter state. Child button calls parent's increment callback.
- DEMO_CODE: Show callback prop pattern
- Explanation: React passes functions as props for child-to-parent communication. Vue uses `defineEmits` + `emit('event-name')`. Both achieve same result — React is more explicit, Vue has event naming conventions.

**ComponentVModel.tsx (COMP-03):**

- Define inline child `function CustomInput({ value, onChange }: { value: string; onChange: (v: string) => void })` mimicking v-model.
- Demo: Parent has `const [text, setText] = useState('')`. CustomInput receives value + onChange. Show the "controlled component" pattern as React's equivalent of v-model.
- DEMO_CODE: Show controlled component pattern (value + onChange)
- Explanation: Vue's `defineModel()` creates two-way binding automatically. React requires explicit value + onChange (controlled pattern). Vue v-model is syntactic sugar for `:modelValue` + `@update:modelValue`.

**FallthroughAttributes.tsx (COMP-04):**

- Define inline child `function MyButton({ label, ...rest }: { label: string } & React.ButtonHTMLAttributes<HTMLButtonElement>)` that spreads `...rest` onto `<button>`.
- Demo: Parent passes className, onClick, disabled to MyButton. Show that extra attrs pass through via spread.
- DEMO_CODE: Show ...rest spread pattern
- Explanation: Vue auto-forwards unrecognized attrs to root element (inheritAttrs). React requires explicit `...rest` spread. Vue can opt out with `inheritAttrs: false`. React always requires explicit forwarding.

**Slots.tsx (COMP-05):**

- Define inline child `function Card({ children, header, footer }: { children: React.ReactNode; header?: React.ReactNode; footer?: React.ReactNode })` demonstrating default slot (children) and named slots (header/footer as props).
- Demo: Use Card with children content, header prop, and footer prop.
- Also show render prop pattern: `function DataList({ renderItem }: { renderItem: (item: string) => React.ReactNode })` for scoped slot equivalent.
- DEMO_CODE: Show children + named props + render props
- Explanation: Vue has `<slot>`, `<slot name="header">`, and scoped slots `<slot :item="item">`. React uses `children` for default, named props for named slots, render props for scoped slots. Different syntax, same composition power.

**ProvideInject.tsx (COMP-06):**

- Create `ThemeContext` with `createContext`. Define ThemeProvider and a deeply nested consumer using `useContext`.
- Demo: Toggle theme between 'light' and 'dark'. Nested component reads theme without prop drilling.
- DEMO_CODE: Show createContext + useContext pattern
- Explanation: Vue uses `provide('key', value)` in parent and `inject('key')` in child. React uses `createContext` + `Provider` + `useContext`. Both solve prop drilling — Vue's API is simpler, React requires more boilerplate but is more type-safe.

**AsyncComponents.tsx (COMP-07):**

- Demo: A button that triggers lazy loading of a "heavy" component using `React.lazy()` + `<Suspense>`. Create an inline module simulation — define a separate component and wrap with `lazy()`.
- Since we can't truly demonstrate async loading in a single file, create a simulated delay: define the component in a separate const, use `React.lazy(() => new Promise(resolve => setTimeout(() => resolve({ default: HeavyComponent }), 1000)))` pattern.
- DEMO_CODE: Show React.lazy + Suspense pattern
- Explanation: Vue has `defineAsyncComponent()`, React has `React.lazy()`. Both wrap dynamic `import()` and need a loading fallback. Vue's `<Suspense>` and React's `<Suspense>` work similarly.
