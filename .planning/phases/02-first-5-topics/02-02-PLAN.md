**Topic 1: Template vs JSX (ESS-01)**

Both sides demonstrate the SAME UI behavior but using framework-native syntax. Demo scenario: a user profile card that shows/hides details and renders a list of skills.

Vue side (`TemplateSyntax.vue`):

- `<script setup lang="ts">` with `ref` imports
- State: `showDetails = ref(false)`, `skills = ref(['Vue', 'TypeScript', 'CSS'])`
- Demo section with border/padding:
  - `<h2>` title "Template Syntax"
  - A profile card showing a name via `{{ name }}`
  - Toggle button: `<button @click="showDetails = !showDetails">` with text "Show/Hide Details"
  - Conditional rendering: `<div v-if="showDetails">` showing email and bio
  - List rendering: `<ul><li v-for="skill in skills" :key="skill">{{ skill }}</li></ul>`
  - Dynamic class binding: `:class="{ 'font-bold': showDetails }"` on the toggle button
- Explanation section below demo (per D-08): A `<div class="mt-6 p-4 bg-slate-50 rounded">` containing:
  - `<h3 class="text-sm font-semibold mb-2">Key Differences</h3>`
  - Short paragraphs (per D-07): Vue uses `<template>` with directives (`v-if`, `v-for`, `v-bind`, `{{ }}`). Directives are compiled away at build time. The template is declarative and closer to HTML.

React side (`TemplateSyntax.tsx`):

- `import { useState } from 'react'`
- Same state: `showDetails`, `skills` array
- Same UI but using JSX: `{showDetails && <div>...</div>}`, `{skills.map(s => <li key={s}>{s}</li>)}`, `className={showDetails ? 'font-bold' : ''}`
- Same toggle button with `onClick`
- Explanation: React uses JSX (JavaScript XML). Conditional rendering uses JS expressions (`&&`, ternary). List rendering uses `.map()`. No special directives -- everything is just JavaScript.

Styling for both: Use Tailwind classes. Demo area: `border border-slate-200 rounded-lg p-4 mb-4`. Buttons: `px-3 py-1 bg-blue-500 text-white rounded hover:bg-blue-600 text-sm`. Title: `text-lg font-semibold mb-3`.

**Topic 2: Reactivity (ESS-02)**

Demo scenario: A counter with increment/decrement + a text input that shows the value in real-time.

Vue side (`Reactivity.vue`):

- `<script setup lang="ts">` with `import { ref, reactive } from 'vue'`
- State: `count = ref(0)`, `user = reactive({ name: '', email: '' })`
- Demo: Counter with +/- buttons showing `{{ count }}`. Two inputs bound with `v-model` to `user.name` and `user.email`. Display: "Hello, {{ user.name }}" updates in real-time.
- Explanation: Vue uses `ref()` for primitives and `reactive()` for objects. Accessed via `.value` in script, auto-unwrapped in template. Vue tracks dependencies automatically -- any template expression using a ref re-renders when that ref changes.

React side (`Reactivity.tsx`):

- `import { useState } from 'react'`
- State: `const [count, setCount] = useState(0)`, `const [name, setName] = useState('')`, `const [email, setEmail] = useState('')`
- Same UI: Counter with `onClick={() => setCount(c => c + 1)}`. Inputs with `value={name} onChange={e => setName(e.target.value)}`.
- Explanation: React uses `useState` returning `[value, setter]` tuple. State is immutable -- you call the setter with a new value. Each setter call triggers a re-render of the component. No `.value` needed but you must always use the setter (direct mutation won't trigger re-render).

Styling: Same pattern as TemplateSyntax. Inputs: `border border-slate-300 rounded px-2 py-1 text-sm`. Counter display: `text-2xl font-bold`.

All demos are self-contained (per D-10). Export as default function (React) / no explicit export needed (Vue SFC).

**Topic: Computed / useMemo (ESS-03)**

Demo scenario: A shopping list where user adds items with prices. A computed/memoized value shows the total, item count, and most expensive item -- all derived from the list and updating automatically.

Vue side (`Computed.vue`):

- `<script setup lang="ts">` with `import { ref, computed } from 'vue'`
- State: `items = ref<{name: string, price: number}[]>([{ name: 'Apple', price: 1.5 }, { name: 'Bread', price: 3.0 }])`, `newName = ref('')`, `newPrice = ref(0)`
- Computed values:
  - `total = computed(() => items.value.reduce((sum, i) => sum + i.price, 0))`
  - `itemCount = computed(() => items.value.length)`
  - `mostExpensive = computed(() => items.value.length ? items.value.reduce((max, i) => i.price > max.price ? i : max).name : 'None')`
- Demo: Form to add item (name input + price input + "Add" button). List of items. Summary box showing Total, Count, Most Expensive.
- Explanation: `computed()` creates a cached derived value. It tracks its reactive dependencies and only recalculates when those dependencies change. Unlike a plain function call in the template, `computed` caches the result -- calling it multiple times without dependency changes returns the cached value.

React side (`Computed.tsx`):

- `import { useState, useMemo } from 'react'`
- State: `const [items, setItems] = useState([...])`, `const [newName, setNewName] = useState('')`, `const [newPrice, setNewPrice] = useState(0)`
- Memoized values:
  - `const total = useMemo(() => items.reduce((sum, i) => sum + i.price, 0), [items])`
  - `const itemCount = useMemo(() => items.length, [items])`
  - `const mostExpensive = useMemo(() => items.length ? items.reduce((max, i) => i.price > max.price ? i : max).name : 'None', [items])`
- Add item: `setItems(prev => [...prev, { name: newName, price: newPrice }])`
- Same UI layout as Vue side.
- Explanation: `useMemo(fn, deps)` memoizes a computed value. Unlike Vue's `computed()` which auto-tracks dependencies, React requires an explicit dependency array `[items]`. If you forget a dependency, the memoized value won't update. `useMemo` is an optimization -- React could recalculate without it, but it avoids expensive recomputations.

Styling: Follow same pattern established in Task 1. Summary box: `bg-blue-50 rounded p-3`. Form inputs inline with "Add" button.

Follow existing file patterns: default export for React, SFC for Vue. Self-contained demos (per D-10).
