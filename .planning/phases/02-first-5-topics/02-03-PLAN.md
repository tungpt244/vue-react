**Topic 1: Watchers / useEffect (ESS-08)**

Demo scenario: A search input that triggers a "debounced API call" (simulated with setTimeout). As user types, a log panel shows when the watcher/effect fires and what value it received. Also a "question" input where typing a question ending with "?" triggers an automatic answer.

Vue side (`Watchers.vue`):

- `<script setup lang="ts">` with `import { ref, watch, watchEffect } from 'vue'`
- State: `question = ref('')`, `answer = ref('Ask a question ending with ?')`, `searchQuery = ref('')`, `logs = ref<string[]>([])`
- `watch(question, (newVal, oldVal) => { logs.value.push(\`watch: "${oldVal}" -> "${newVal}"\`); if (newVal.endsWith('?')) { answer.value = 'Thinking...'; setTimeout(() => answer.value = 'Yes! 42 is the answer.', 500) } })`
- `watchEffect(() => { if (searchQuery.value) { logs.value.push(\`watchEffect: searching "${searchQuery.value}"\`) } })`
- Demo: Two inputs (question, search), answer display, scrollable log panel showing watcher triggers
- Clear logs button
- Explanation: `watch(source, callback)` explicitly watches a specific ref and provides old/new values. `watchEffect()` auto-tracks any reactive dependency accessed inside it -- no need to specify what to watch. Vue watchers are synchronous by default (flush: 'pre'). Both are automatically cleaned up when the component unmounts.

React side (`Watchers.tsx`):

- `import { useState, useEffect, useRef } from 'react'`
- State: `question`, `answer`, `searchQuery`, `logs` (all useState)
- `useEffect(() => { logs setter to append; if (question.endsWith('?')) { setAnswer('Thinking...'); const t = setTimeout(() => setAnswer('Yes! 42 is the answer.'), 500); return () => clearTimeout(t) } }, [question])`
- Separate `useEffect` for searchQuery
- Same UI as Vue side
- Explanation: React has only `useEffect(fn, deps)` -- no separate watch vs watchEffect. The dependency array `[question]` is like `watch(question, ...)`. An empty `[]` runs once on mount. Returning a cleanup function from `useEffect` handles teardown. Unlike Vue's `watchEffect`, React doesn't auto-track dependencies -- you must list them explicitly.

**Topic 2: Lifecycle Hooks (ESS-10)**

Demo scenario: A timer component that starts on mount and cleans up on unmount. A toggle button shows/hides the timer to demonstrate mount/unmount. A log panel tracks lifecycle events.

Vue side (`Lifecycle.vue`):

- `<script setup lang="ts">` with `import { ref, onMounted, onUnmounted, onBeforeMount, onBeforeUnmount } from 'vue'`
- Create a child component inline using `defineComponent` or just use the main component:
- State: `showTimer = ref(true)`, `elapsed = ref(0)`, `logs = ref<string[]>([])`
- Use a separate inner component `TimerWidget` (defined in same file with `defineComponent`) that:
  - `onBeforeMount(() => logs push 'beforeMount')`, `onMounted(() => { logs push 'mounted'; start interval })`, `onBeforeUnmount(() => logs push 'beforeUnmount')`, `onUnmounted(() => { logs push 'unmounted'; clear interval })`
  - Shows elapsed seconds
- Actually, simpler approach: use a single component. Show/hide a section that demonstrates the concept.
- Main component: `showTimer` toggle. When `showTimer` is true, display a timer section. Use `onMounted` to start an interval updating `elapsed` every second. Use `onUnmounted` to clear it. Log each lifecycle event to `logs`.
- BUT: onMounted/onUnmounted fire once for the main component, not on toggle. To show mount/unmount cycles, need a child component.
- APPROACH: Create the timer as the topic component itself. The "show/hide" controls a child `<TimerDisplay>` component that has the lifecycle hooks. Define it in the same SFC using a second `<script>` block or just reference it.
- Simplest: Define a separate `const TimerDisplay` component using `defineComponent` inside `<script setup>`, then conditionally render it.

Implementation:

```vue
<script setup lang="ts">
import { ref, defineComponent, h, onMounted, onUnmounted } from 'vue'

const logs = ref<string[]>([])
const showTimer = ref(true)

const TimerDisplay = defineComponent({
  setup() {
    const elapsed = ref(0)
    let interval: ReturnType<typeof setInterval>
    onMounted(() => {
      logs.value.push(`[${new Date().toLocaleTimeString()}] onMounted`)
      interval = setInterval(() => elapsed.value++, 1000)
    })
    onUnmounted(() => {
      logs.value.push(`[${new Date().toLocaleTimeString()}] onUnmounted`)
      clearInterval(interval)
    })
    return () => h('div', { class: 'text-2xl font-bold' }, `${elapsed.value}s`)
  },
})
</script>
```

- Demo: Toggle button, TimerDisplay (conditional), log panel
- Explanation: Vue provides explicit lifecycle hooks: `onMounted`, `onUnmounted`, `onBeforeMount`, `onBeforeUnmount`, etc. Each maps to a specific phase. They only run once per component instance. When the component is toggled off and on, a new instance is created and all hooks fire again.

React side (`Lifecycle.tsx`):

- Define `TimerDisplay` as a function component inside the file:

```tsx
function TimerDisplay({ onLog }: { onLog: (msg: string) => void }) {
  const [elapsed, setElapsed] = useState(0)
  useEffect(() => {
    onLog(`[${new Date().toLocaleTimeString()}] mounted (useEffect)`)
    const interval = setInterval(() => setElapsed((e) => e + 1), 1000)
    return () => {
      onLog(`[${new Date().toLocaleTimeString()}] unmounted (cleanup)`)
      clearInterval(interval)
    }
  }, [])
  return <div className="text-2xl font-bold">{elapsed}s</div>
}
```

- Main component: `showTimer` state, toggle button, conditional `{showTimer && <TimerDisplay onLog={...} />}`, log panel
- Explanation: React has no separate lifecycle hooks. `useEffect` with `[]` deps = `onMounted`. The cleanup function returned from `useEffect` = `onUnmounted`. React combines mount + unmount into a single `useEffect` call. There's no `onBeforeMount` equivalent -- effects run after render, not before.

Both: Clear logs button. Log panel: `bg-slate-900 text-green-400 font-mono text-xs p-3 rounded max-h-40 overflow-y-auto`.

**Topic: Class & Style Bindings (ESS-11)**

Demo scenario: A theme switcher that applies different classes and inline styles dynamically. User can toggle between themes, toggle individual CSS classes on/off, and adjust inline style values (font size, color) via inputs.

Vue side (`ClassStyleBindings.vue`):

- `<script setup lang="ts">` with `import { ref, computed } from 'vue'`
- State: `isActive = ref(true)`, `isHighlighted = ref(false)`, `fontSize = ref(16)`, `textColor = ref('#334155')`, `theme = ref<'light' | 'dark'>('light')`
- Demo section 1 -- Object syntax for `:class`:
  - A box with `:class="{ active: isActive, highlighted: isHighlighted }"` where `active` and `highlighted` are Tailwind-compatible classes defined in a `<style scoped>` block or just using Tailwind directly
  - Actually, use Tailwind: `:class="{ 'bg-blue-100 border-blue-500': isActive, 'ring-2 ring-yellow-400': isHighlighted }"`
  - Toggle buttons for `isActive` and `isHighlighted`
- Demo section 2 -- Array syntax for `:class`:
  - `<div :class="[theme === 'dark' ? 'bg-slate-800 text-white' : 'bg-white text-slate-800', 'p-3 rounded border']">Theme preview</div>`
  - Buttons to switch theme
- Demo section 3 -- `:style` binding:
  - `<p :style="{ fontSize: fontSize + 'px', color: textColor }">Styled text</p>`
  - Range input for fontSize (12-32), color input for textColor
- Explanation: Vue has dedicated `:class` and `:style` directives that accept objects, arrays, or strings. Object syntax `{ className: condition }` is the most common. Array syntax allows combining static and dynamic classes. `:style` auto-prefixes CSS properties. Vue also supports binding to `class` and `style` on components (falls through to root element).

React side (`ClassStyleBindings.tsx`):

- `import { useState } from 'react'`
- Same state, same demo sections
- Section 1 -- className with template literals or clsx-style concatenation (no clsx import needed, use template literals):
  - `className={\`p-3 rounded border \${isActive ? 'bg-blue-100 border-blue-500' : ''} \${isHighlighted ? 'ring-2 ring-yellow-400' : ''}\`}`
  - Same toggle buttons
- Section 2 -- className with ternary:
  - `className={\`\${theme === 'dark' ? 'bg-slate-800 text-white' : 'bg-white text-slate-800'} p-3 rounded border\`}`
- Section 3 -- style object:
  - `style={{ fontSize: \`\${fontSize}px\`, color: textColor }}`
  - Same range/color inputs
- Explanation: React uses `className` (not `class`) and `style` as a JS object (not a string). `className` is always a string -- no built-in object/array syntax like Vue's `:class`. For complex class logic, libraries like `clsx` help. `style` uses camelCase properties (`fontSize` not `font-size`) and string/number values. React doesn't auto-prefix CSS properties.

Both sides: Each demo section has a small label/heading. Consistent Tailwind styling.
